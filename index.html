Сборка

Кратко

Для упрощения процесса сборки и конфигурации системы был написан скрипт build.py. Он принимает следующие три команды в качестве аргументов командной строки:

  • verify-env - проверяет системные пакеты, символические ссылки и зависимости.
  • setup-env - устанавливает системные пакеты, символические ссылки и зависимости. Необходимо запускать от имени администратора. (Только для Ubuntu Linux)
  • build - собирает библиотеку в релизном режиме.

Стандартный порядок сборки библиотеки таков:

  1. Устанавливаем все необходимые pip-пакеты. Для этого в корне проекта выполняем команду: pip install -r requirements.txt
  2. Проверяем системные пакеты, символические ссылки и зависимости ./build.py verify-env

Если были сообщения об ошибках, то выполняем конфигурацию системы автоматически или вручную. Для автоконфигурации выполняем команду: sudo ./build.py setup-env

[!TIP] Все необходимые пакеты можно установить вручную.Их список можно вывести командой verify-env.

  1. Сборка библиотеки

    Примеры:

./build.py # собирает CUDA-версию со стандартными опциями
./build.py build # псевдоним для команды выше
./build.py --debug # собирает версию для отладки
./build.py --no-cuda # собирает FFTW-версию
./build.py --native-pybind --no-cuda # собирает FFTW-версию с системным pybind11

[!TIP] Команды build.py build и build.py - синонимичны.

Команда build принимает следующие опции:

АргументПротивоположный аргументПо умолчаниюОписание
--shared--static**--shared**Динамическая или статическая библиотека
--test--no-test**--no-test**Сборка тестов (Google Test)
--cuda--no-cuda**--cuda**Сборка CUDA-DSP
--python-bindings--no-python-bindings**--python-bindings**Сборка биндингов для Python 3
--release--debug**--release**Релизная или отладочная версия
--native-pybind--builtin-pybind**--builtin-pybind**Использовать системный pybind11 или встроенный
  1. После успешной сборки библиотеки в папке py3 появится нужный файл модуля Python.

Требования

  • Python версии 3.7 и выше. Доступен режим совместимости с Python 3.5. (необходимы тесты).
  • CMake версии 3.20 и выше.

  • NVIDIA CUDA Toolkit в системном пути поиска библиотек.

    [!NOTE]

    Если при сборке возникают ошибки, связанные с невозможностью задания стандарта CUDA17 для > CUDA-файлов проекта, обновите cmake.

Рекомендации

  1. Добавить директорию с nvcc в системный PATH, например:

    export PATH="/usr/local/cuda-12.2/bin:$PATH"  # для записи в PATH
echo "export PATH=\"/usr/local/cuda-12.2/bin:$PATH\"" >> ~/.profile   
echo "export PATH=\"/usr/local/cuda-12.2/bin:$PATH\"" >> ~/.bashrc    # для сохранения после перезагрузки
  2. Установить следующие apt-пакеты:
    • build-essentials
    • python3, python3-dev
    • cmake
    • pybind11 - если использовать --native-pybind

Полезные команды

Запуск скрипта Python в режиме отладки:

gdb
file python3 
run main_bp_<ИМЯ СКРИПТА>.py -f m1_<ФАЙЛ> <АРГУМЕНТЫ>
Вывод стактрейса при системном сигнале:
bt 3 # количество функций в стактрейсе
Вывод локальных переменных и аргументов функций:
bt 3           # получаем стактрейс
select-frame 1 # выбираем функцию для вывода
info locals    # выводим локальные переменные
info args      # выводим аргументы

API

Проект предоставляет публичные независимые заголовочные файлы для разработчиков фронтендов! Их можно найти в директории include/quasar/api внутри проекта.

  • quasar/api/span.h - платформонезависимый std::span-контейнер.
  • quasar/api/image.h - API для работы с радиолокационными изображениями.
  • quasar/api/format.h - форматтеры для сериализуемых типов.
  • quasar/api/dsp.h - API для работы с ЦОС.
  • quasar/api/checksum.h - API для работы с контрольными суммами.

Для их использования должен поддерживаться хотя бы C++17.
